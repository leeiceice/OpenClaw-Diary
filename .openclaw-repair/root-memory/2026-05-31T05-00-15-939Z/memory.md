## 轨迹挖掘 v3 提炼（2026-05-17）

### Cron 管理教训（最高频领域，22条记录）
- **Cron 定义维护**：message 长度控制 < 80 字符避免上下文膨胀（有案例从 406→49 字符，tokens 减少 99.94%）
- **Timeout 设置**：对话类任务 timeout ≥ 300s，避免 120s 超时误判
- **失败重试**：首次失败等待 15min 冷却期再重试，避免连环重试
- **生命周期**：一次性任务完成即删除 cron definition，避免残留

### Memory 系统规范
- MEMORY.md 版本号永远在最后一行，新增内容插入版本号上方
- 每日提炼合并精简，不重复
- corrections.md 中的错误记录可删除原记录（不影响提炼结果）

### 内容工作流
- 浏览器抓取微信公众号文章需准备备用方案（镜像/直接粘贴）
- Ontology 管道 timeout 从 120s→300s 可减少超时失败

## 05/18~05/24 新教训

### 行动边界与任务确认
- **收到「做」≠ 授权执行**：先说理解 → 分解步骤 → Lee 确认 → 执行（红线写入 SOUL.md）
- **Skill 安装**：必须走安全审查流程，不得跳过确认直接执行（已多次漏过）

### 微信读书 / weread
- `.env` 格式要求：每个 KEY 必须独立一行，连在一起会导致解析错误（如 `FEISHU_BOT_TOKEN=xxxDEEPSEEK_API_KEY=sk-xxx` 会导致 DEEPSEEK_API_KEY 拿到错误值）
- **微信读书书名自动查找**：无 bookId 时，用 `/store/search?scope=10` 搜索书名获取 bookId，再拉热门划线
- **DeepSeek API**：model 名应为 `deepseek-v4-flash`（非 `deepseek-flash`，后者报 400）
- **DeepSeek 填充安全约束**：无内容写「（当日未记录）」，不自行编造

### 沟通效率
- **高效沟通三层区分**：Fact（可验证）→ Inference（因果推断）→ Judgment（评价），避免混用
- **反信息茧房**：推荐内容主动说「主流反面是…」，单一时提醒

### 系统健康
- 安全群 chat_id 无效（`oc_1f77586fc34cdac8f43a4e9733eafc`），推送失败，需 Lee 确认
- HEARTBEAT 新增向量搜索健康检查（L2 Scene 反馈回路）

### Git / 协作
- `git pull --rebase` 代替普通 pull，避免多余 merge commit
- 分叉（diverged）→ 立即 rebase → 再 push
