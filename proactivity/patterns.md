# Proactive Patterns — 可复用的主动行为模式

> 从实际执行中提炼的主动行为模板。

---

## 模式 1：系统资源检查（每次长对话后）

**触发**：对话超过 20 分钟  
**动作**：
```
- 检查内存使用率（>80% 则清理）
- 检查 openclaw cron 服务状态
- 检查关键端口（8080/8090等）
- 检查 agent_webhook.py 是否运行
```

---

## 模式 2：记忆容量检查（每次写入前）

**触发**：主动写入任何记忆文件  
**动作**：
```
- 检查目标文件大小/行数
- 若 >80% 容量 → 先整理再写入
```

---

## 模式 3：架构稳定性检查（每周一次）

**触发**：每周一生成话题卡时顺便执行  
**动作**：
```
- ps aux 扫描异常进程
- ss -tlnp 检查关键端口
- openclaw cron list 检查调度状态
```

---

## 模式 4：会话结束摘要

**触发**：对话结束时（判断不再回复）  
**动作**：
```
- 更新 session-state.md（结论/进行中/下一步）
- 重要发现 → 写入 MEMORY.md 或 proactivity/log.md
- 容量 >80% → 立即整理
```

---

## 模式 5：外部推送处理（收到 webhook/群消息）

**触发**：收到小马/CC 的 HTTP 推送，或群里有收藏通知  
**动作**：
```
- 从 GitHub 拉取最新 Obsidian（git pull）
- 执行关联分析（association_analysis.py）
- 话题匹配，判断是否需要推送飞书群
- 更新收藏索引
```
