## 🔑 本周学到的重要知识（系统级）

### 1. Cron model 管理红线
- 所有 cron 必须用 allowlist 备案的 model（`deepseek/deepseek-v4-flash`）
- 创建时就要确认，修改用 `cron update` 工具（**不用直接 edit jobs.json**）
- Cron 区别：主会话用 M3 / cron 用 DeepSeek Flash

### 2. API key 验证流程
- **先 curl 验证 key+endpoint，再动配置**
- Kimi For Coding 是「编程 CLI 专用」，普通 Agent 不能接入
- Anthropic transport 必须显式 `maxTokens > 0`

### 3. 模型评估方法论
- 计算题 + 代码题 = 高区分度；创意题 / 摘要 = 低区分度
- **互评去自评机制**：剔除作者自评，3 个非作者裁判更可靠
- V4-Flash 性价比远超 V4-Pro（自动评分第一，双盲 95%）

### 4. 飞书推送路由原则（06-03 Lee 授权）
- 任务性质路由：技术成果/架构/HEARTBEAT → 进化群
- Lee 行为（喝水/GNC/学习）→ 日常安排群
- 外部信息（新闻/书/文章）→ 新闻群/内容收集群
- 系统安全/数据（备份/审计/异常）→ 安全群

### 5. 记忆即系统
- L0 是实时写入，不是 cron 补跑
- 每次重要操作后必须写 memory/
- 「我以为记了，实际上没记」是最危险的状态

### 6. 推理题设计陷阱
- 「找解型」数字谜题比「推理唯一结论」更可验证
- 多解问题导致评测对象修正，浪费时间
