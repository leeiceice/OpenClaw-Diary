## 23:58 cron-chat-id-audit-daily 失败修复

**症状**：「Cron job cron-chat-id-audit-daily failed 2 times」+ 错误 `Channel is required when multiple channels are configured: lightclawbot, feishu, openclaw-weixin. Pass --channel <channel> to choose one.`

**根因**：cron delivery 配 `mode: announce + channel: "last"`，但当前 session 有 3 个 channel（lightclawbot/feishu/openclaw-weixin）→ "last" 解析失败 → 推不出去。脚本**本身跑通了**（24h 去重命中，无异常），只是 delivery 报错。

**修法**：delivery 改 `mode: none`（脚本内部 `openclaw message send` 自推 → 完整控制推送）。沿用修「轨迹挖掘」「newsletter」同一套路。

**实证 MEMORY 6/3 「沉默失败 4 case」**：

- 推论：cron 健康度 = `status: ok` && `delivery: delivered` 才算真活
