# 向量搜索系统架构

> 小龙虾的混合检索系统：BM25 × Qwen3-Embedding-4B 向量，RRF 融合 + ECC 置信度叠加。

## 一、定位

三层记忆优先级的**第二层**（中期 / 长期召回层），介于"当前会话上下文"与"联网搜索"之间。

**为什么不只用关键词？** — 同义词、概念、隐含意图召回率差。
**为什么不只用向量？** — 专有名词、版本号、代码片段召回率低。

## 二、架构总览

```
                    ┌──────────────────────────────┐
                    │   memory_search.py (主入口)  │
                    └──────────┬───────────────────┘
                               │
                ┌──────────────┴──────────────┐
                │                             │
        ┌───────▼────────┐          ┌─────────▼─────────┐
        │  BM25 关键词层  │          │  Qwen3-Embedding  │
        │  (快速/精确词)  │          │  -4B 向量层       │
        └───────┬────────┘          │  (语义召回)        │
                │                    └─────────┬─────────┘
                │                              │
                └──────────┬───────────────────┘
                           │
                  ┌────────▼─────────┐
                  │   RRF 融合排序   │
                  │  (Reciprocal     │
                  │ Rank Fusion)     │
                  └────────┬─────────┘
                           │
                  ┌────────▼─────────┐
                  │  L2 置信度叠加   │
                  │  (ECC 框架)      │
                  └──────────────────┘
```

## 三、数据流（事实层）

| 步骤 | 输入 | 处理 | 输出 |
|------|------|------|------|
| 1. 入库 | MEMORY.md / memory/*.md / 日记 raw | Qwen3-Embedding-4B 编码 | 向量 → `data/vector_store.db` |
| 2. 召回 1 | 用户 query | BM25 关键词打分 | top-N 文档 + 分数 |
| 3. 召回 2 | 用户 query | 向量相似度检索 | top-N 文档 + 分数 |
| 4. 融合 | 两个 top-N | RRF 公式合并 | 重排序结果 |
| 5. 置信度 | 排序结果 | ECC 算法（基础30% + 验证加成 - 失败惩罚） | 带置信度标签的记忆 |

## 四、ECC 置信度反馈回路

### 四状态机

| 状态 | 区间 | 含义 |
|------|------|------|
| 猜测 | <30% | 未经验证的假设 |
| 直觉 | 30-59% | 推理一致但未实测 |
| 经验 | 60-84% | 至少一次实测验证 |
| 知识 | ≥85% | 多源验证 / 长期无反驳 |

### 评分公式

```
基础分: 30%
+ 实测验证来源: +20%
+ 修复成功: +15%（最高 70%，需经验级才能到知识级）
+ 失败: -20%（最低 10%）
```

### 反馈触发

- 心跳检查发现异常 → 记录为观察条目（实际写 `self-improving/corrections.md`，**2026-06-07 修正**）
- 修复后实测 → 更新置信度
- ≥60% 触发 `/evolve` 聚类
- ≥85% 触发 `/promote` 技能化

## 五、关键模块路径

| 模块 | 路径 | 职责 |
|------|------|------|
| 主搜索 | `scripts/memory_search.py` | BM25 × vector RRF 融合入口 |
| 嵌入 | Qwen3-Embedding-4B（外部 API） | 文本 → 768 维向量 |
| 向量库 | `data/vector_store.db` | SQLite 存储 |
| 记忆源 | `MEMORY.md` + `memory/*.md` + `~/Obsidian/日记/raw/*.md` | 唯一真相源 |
| L2 反馈入口 | `memory/deviations/scenes/` | 场景块反写置信度 |
| 纠正日志 | `self-improving/corrections.md` | 错误记录 → 影响评分（2026-06-07 修正） |
| 快照 | `proactivity/intuition-snapshot.json` | 每日置信度分布快照 |

## 六、已知约束

1. **embedding 模型固定 Qwen3-Embedding-4B**（2026-05-18 升级后稳定）
2. **RRF 融合权重**：BM25 与 vector 各占 0.5（可调）
3. **离线降级**：向量库不可用时，自动降级纯 BM25
4. **时区语义**：内部 UTC 存储，显示 CST（Asia/Shanghai，UTC+8）
5. **token 预算**：每次返回 top-5 + 摘要，避免上下文膨胀
6. **可逆性优先**：所有写入必须可回滚（git 提交 + frontmatter 历史）

## 七、运行时检查（HEARTBEAT 第三步 B）

每次心跳会验证：

- ✅ 向量索引 24h 内是否有更新（→ 是否需要重嵌入）
- ✅ L2 scene 块是否回写
- ✅ corrections.md 是否有新增（→ L2 反馈活跃度）
- ✅ BM25 × vector RRF 是否正常（vs 降级纯 BM25）
- ✅ confidence 分布快照是否存在（24h 内）

## 八、调用模式（给使用者的提示）

```python
# 1. 提问前先查记忆
results = memory_search(query="...", top_k=5)

# 2. 三层记忆优先级
#   长期 (MEMORY.md) > 中期 (memory/*.md) > 当前会话 > 联网
```

**判定规则：**
- 置信度 < 60% → 不用于关键决策，必须先验证或询问 Lee
- 置信度 60-84% → 可辅助推理，建议二次确认
- 置信度 ≥ 85% → 可直接使用，仍保留可证伪标记
